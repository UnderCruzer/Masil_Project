# 마실 — Design Document v1.0

*FRD v1.0 (2026-05-23) 반영 업데이트*

호스피스·완화치료 환자를 위한 AI 맞춤 소원여행 플래너.
따뜻하고 안심되는 분위기를 기반으로, 의료적 제약 속에서도 여행의 설렘을 느낄 수 있도록 설계한다.

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **안심감 (Reassurance)** | 차갑지 않고 따뜻한 색조. 버튼·입력 요소에 충분한 여백과 큰 터치 영역 |
| **명확성 (Clarity)** | 단계별 입력 흐름, 짧은 문장, 키워드 강조로 인지 부담 최소화 |
| **원페이지 구성 (One Page, One Thing)** | 토스 스타일: 화면당 하나의 질문만. 복잡한 조건 입력도 한 항목씩 분리하여 부담 없이 진행 |
| **쉬운 언어 (Plain Language)** | 환자·보호자 대상 앱. 의학 용어·수치 대신 일상 언어 사용. "체력 3단계" 대신 "아침에 일어나기 힘들다" |
| **접근성 (Accessibility)** | 무장애 여행 앱답게 충분한 컬러 대비, 터치 타깃 최소 48dp |
| **모션 (Motion)** | 200–350ms AnimatedContainer / AnimatedSwitcher로 부드러운 전환, 과도한 애니메이션 지양 |

## 4. 화면 흐름 (Screen Flow)

```
Splash (2.2s)
    │ fadeIn 700ms
    ▼
Onboarding (4페이지 PageView)
    │ 건너뛰기 / 소원여행 시작하기
    ▼
[Condition — 원페이지 스텝 구성 (토스 스타일)]
  Step 1: 이동 가능 거리   (프리셋 칩 3종)
  Step 2: 이동 보조기기    (단일 선택 칩)
  Step 3: 투약 시간대      (복수 선택 칩 + 직접 지정 옵션)
  Step 4: 체력 수준        (5단계 바 + 일상 언어 설명)
  Step 5: 동행 정보        (인원·보호자·차량·비상연락처)
    │ 다음
    ▼
[Wish — 원페이지 스텝 구성]
  Step 1: 테마 카드 선택   (복수 선택 그리드)
  Step 2: 소원 자유 입력   (멀티라인 텍스트)
  Step 3: 희망 지역 선택   (Wrap 칩, 복수 선택)
  Step 4: 여행 날짜 선택   (캘린더 date picker)
    │ AI 소원 플랜 생성
    ▼
Plan Loading
  └─ 지역 관광지 미리보기 카드 (TourAPI 캐러셀)
  └─ 진행 메시지 순환 + 진행률 표시
    │
    ▼
Plan Result (AI 여행 플랜)
  ├─ 타임라인 카드 (TourAPI 이미지 포함)
  ├─ 투약 알림 가드 (오렌지 강조)
  ├─ 일몰 안전 가드
  ├─ 🗺️ 네이버 지도 경로 보기 (폴백 처리 포함)
  ├─ 🏥 주변 의료시설 (심평원 API)
  └─ 📤 가족과 일정 공유 (Firestore + 카카오톡)
    │
    ├─ [AI 재조정] ◄── 여행 당일 아침 푸시 알림
    │   └─ Condition Check-in
    │       ├─ 😊 좋음 → 플랜 유지, 출발
    │       ├─ 😐 보통 → 장소 카드 목록 → 제외할 장소 선택 → 재생성
    │       └─ 😔 나쁨 → 이동거리 단축 경량 플랜 재생성
    │
    └─ [타임캡슐 남기기]
        └─ Time Capsule
            ├─ 장소 선택 (스팟 목록, 최대 3개)
            ├─ 🎤 음성 메시지 (최대 2분)
            ├─ ✉️ 편지 작성
            ├─ 📷 사진 추가
            └─ 열람 가족 지정 + Geofencing 잠금

[플레이스홀더 / 미구현]
  ├─ 에너지 버블 맵 (SCREEN 02, 플랜 화면 내 통합 예정)
  ├─ 동행·시설·숙박 예약
  └─ AI 포토북
```

## 5. 컴포넌트 라이브러리

### 5-1. 기본 버튼

**Primary ElevatedButton**
- 배경: `primary` / 비활성: `gray200`
- 높이: 52–54dp
- 반경: 12–16dp
- 전체 너비 (`double.infinity`)
- 하단 컨테이너에 위에서 오는 그림자 (`boxShadow offset (0,-4)`) 포함

**텍스트 버튼**
- 건너뛰기 등 보조 액션
- 색상: `gray400`

### 5-2. 이동거리 프리셋 칩 (F01-01)

슬라이더 대신 텍스트 프리셋 제공. 수치를 직접 판단하기 어려운 환자·보호자를 고려한 설계.

| 레이블 | 거리 | 아이콘 |
|--------|------|--------|
| 🚶 가까운 마실 | 5km 이내 | `directions_walk_rounded` |
| 🚗 무리 없는 외출 | 10~15km | `directions_car_rounded` |
| 🗺️ 하루 여행 | 20km 이상 | `explore_rounded` |

AnimatedContainer 200ms 전환 (단일 선택, 솔리드 variant).
선택 후 "약 Nkm 이내로 장소를 찾아드릴게요" 안내 표시.
상세 조정이 필요한 경우 슬라이더(1~50km) 보조 제공 (기본 collapsed).

### 5-3. 선택 칩 (단일 선택 — 솔리드)

AnimatedContainer 200ms 전환

| 상태 | 배경 | 보더 | 텍스트/아이콘 |
|------|------|------|---------------|
| 비선택 | `white` | `gray200` 1.5dp | `gray600` |
| 선택 | `primary` | `primary` 1.5dp | `white` + shadow |

사용처: 이동 보조기구 선택, 이동거리 프리셋, 보호자 여부, 차량 여부

### 5-4. 선택 칩 (복수 선택 — 연한 variant)

| 상태 | 배경 | 보더 | 텍스트/아이콘 |
|------|------|------|---------------|
| 비선택 | `white` | `gray200` 1.5dp | `gray400` |
| 선택 | `primaryLight` | `primary` 1.5dp | `primary` + check 아이콘 |

사용처: 투약 시간대, 소원 태그, 희망 지역 (복수 선택)

### 5-5. 체력 수준 바 (5단계 + 일상 언어 설명)

수치 대신 일상 언어로 체력 수준을 표현하여 직관적으로 선택 가능.

| 단계 | 색상 | 설명 텍스트 |
|------|------|-------------|
| 1 | `#E53935` | 입맛이 없고 일어나기 힘들어요 |
| 2 | `#FF7043` | 조금 피곤하고 움직이기 귀찮아요 |
| 3 | `#FFB300` | 보통이에요, 무난하게 다닐 수 있어요 |
| 4 | `#66BB6A` | 컨디션 괜찮아요 |
| 5 | `#2BBFAF` | 컨디션 좋아요! |

가로 정렬, 동일 너비 Expanded × 5.
선택 셀: 단계별 컬러, 체크 아이콘, shadow.
비선택 셀: `gray100`, 단계 숫자.
하단: 12sp 설명 텍스트 (lineHeight 1.4, 2줄 허용).

### 5-6. 투약 시간 입력 (F01-03)

- **프리셋 칩**: 오전 / 오후 / 야간 (복수 선택 연한 variant)
- **시간 직접 지정**: "직접 입력" 탭 시 TimePicker 표시 (시작~종료)
- 선택 시간대: AI 플랜 생성 시 해당 구간 이동·관람 일정 배제

### 5-7. 동행 정보 섹션 (F01-05 확장)

단순 인원수에서 동행 전반의 정보로 확장.

```
동행 인원 (환자 포함): 숫자 카운터 (1~10명)
보호자 동행 여부:      [예] / [아니오] 칩
차량 이동 가능 여부:    [예] / [아니오] 칩
비상 연락처:           전화번호 텍스트 입력 (선택 사항)
```

### 5-8. 캘린더 날짜 선택기 (F03-04)

`showDateRangePicker` (시작일 + 종료일) 또는 단일 날짜 선택.
선택 기간으로 여행 일수 자동 계산. 최대 5일.
선택 시: "N박 N일 여행이네요!" 확인 문구 표시.
미정: "[날짜 미정으로 계속]" 옵션 제공.

### 5-9. 숫자 카운터 (동행 인원·일수)

- `+` / `-` 버튼: 원형 36–40dp, 활성 `primary` / 비활성 `gray200`
- 숫자 표시: 28sp 700 `primary` (AnimatedSwitcher ScaleTransition)

### 5-10. 지역 선택 칩 (복수 선택)

FRD 결론: 복수 선택으로 변경 (n박 m일 여행 시 여러 지역 이동 가능).

Wrap 형태 pill 칩 (r24). 여러 지역 동시 선택 가능.
단일 선택 솔리드 variant 적용 (선택 = `primary` 배경).

### 5-11. 소원 태그 그리드

2열 GridView, aspectRatio 2.6.
왼쪽 이모지(20sp) + 레이블(13sp 600) + 설명(10sp) + 선택 시 check 아이콘.

### 5-12. 카드 (공통)

```
white 배경, borderRadius 14–20dp
border: gray200 (또는 선택 상태에 따라 primary)
boxShadow: black 4–6% opacity, blur 8–12, offset (0,2)
padding: 14–20dp
```

### 5-13. 그라데이션 헤더 카드

`#2BBFAF` → `#1A9488`, topLeft → bottomRight
white / white70 텍스트, r16–20dp
사용처: Wish 안내 배너, Plan 결과 헤더, Condition Check-in 헤더

### 5-14. 타임라인 (플랜 스팟 목록)

- 왼쪽: 28dp 원형 순서 번호 (카테고리 컬러)
- 연결선: width 2, `gray200`, vertical margin 4
- 오른쪽 카드: name + 카테고리 뱃지 + TourAPI 이미지 섬네일 + description + 메타 칩 + 접근성 노트

### 5-15. 투약 알림 카드

배경 `#FFF8E1`, 보더 `#FFCC02` 1dp, r12
아이콘 `#FF8F00`, 텍스트 `#795548`

### 5-16. 주의사항 카드 (여행 전 확인사항)

배경 `secondaryLight`, 보더 `#FFE0B2`, r16
아이콘·타이틀 `secondary`

### 5-17. 로딩 중 관광지 미리보기 카드 (F03-07)

AI 플랜 생성 대기 시간 동안 지역 명소를 소개해 기다림을 자연스럽게 해소.

```
white 카드, r16, shadow
┌─────────────────────────────┐
│ [이미지 80×80 r10]  장소명   │
│                    간단 설명 │
│              ✨ 어떠세요?    │
└─────────────────────────────┘
```

캐러셀 형태 자동 슬라이드 (3초 간격, PageIndicator 표시).
데이터: TourAPI locationBasedList (선택 지역 기준).

### 5-18. 컨디션 이모지 선택 (F04-06)

당일 아침 컨디션을 3단계 이모지로 직관적으로 입력.

```

[😊 좋아요!]   [😐 보통이에요]   [😔 힘들어요]
  초록 테두리    노랑 테두리       빨강 테두리
```

선택 시: 해당 컬러 border + scale 1.1 애니메이션 (200ms).

### 5-19. 네이버 지도 연결 버튼 (F04-04)

폴백 처리 필수 (앱 미설치 시 사용자 경험 보호).

```
    ├─ 앱 설치됨: nmap:// 딥링크로 전체 경로
    └─ 앱 미설치: 바텀시트 표시
        ├─ 🌐 웹에서 경로 보기
        ├─ 📋 주소 복사
        └─ 👥 보호자에게 공유
```

### 6-1. Splash
